## Verification methodology (future branch)

1. For each board, extract claims via `get_figjam`: services, labeled connectors (direction +
   transport), infra resources, queue/topic names, stickies (architect caveats).
2. Diff each claim against the map: connection exists? transport matches? endpoints/queues known?
3. Classify like the drift report: ✅ agrees / 🆕 diagram-only edge / ⚠ map-only edge / ❓ ambiguous.
4. Diagram-sourced corrections enter the dataset as `provenance.source: manual` with
   `evidence: <figma url>` — diagrams drift too; code evidence outranks them (ADR-0007).
