# ADR-0001: Record architecture decisions

**Status:** accepted — 2026-06-09

## Context

The project is being rebooted from a prototype that accreted structure nobody decided on (a MongoDB round-trip for static data, two coexisting app generations). Decisions need to be explicit so they can be challenged instead of rediscovered.

## Decision

Record every significant architecture decision as a short ADR in `docs/adr/`, numbered sequentially: Status, Context, Decision, Consequences.

## Consequences

New contributors (and the Infrastructure/Architecture reviewers) can read why the project is shaped the way it is in ten minutes.
